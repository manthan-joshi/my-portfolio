CodeTaxonomer

A taxon search engine, built by a bench guy.

Web app for searching taxonomic data with filters like 'starts with,' 'contains,' or 'ends with.' Built with FastAPI, SQLModel, and Dash. A bioinformatics final project, done with a friend who actually knows backend.

I'm more comfortable with a pipette than with Python. This project was about fixing that, at least partially, by building something actually usable instead of running one more tutorial.

What it does

The app lets users query a taxonomic dataset through a clean UI. You can filter for names that start with a given string, contain a string, or end with one. Results are returned fast and displayed in an interactive Dash dashboard. It's the kind of utility a bench biologist actually reaches for when they're trying to find an organism by partial name.

The stack

  • FastAPI for the backend API layer
  • SQLModel for the data model and database interaction
  • Dash for the frontend UI and interactive results display
  • Deployed as a working demo with code on GitHub

What I learned, honestly

I had a friend with a bioinformatics background help me get the architecture right and debug the parts that felt like magic to me. That's not a thing to hide. Knowing what you don't know and asking good questions is the bioinformatics skill that actually matters.

The code is on GitHub if you want to look at it. The README is honest about what I wrote and what got refined with help.

A bench scientist who can build a working web app is more useful than one who pretends to.
